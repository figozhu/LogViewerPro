# 全局配置

## 核心行为准则

### 1. 思维与响应语言设置
- **思维语言**: 始终使用英文进行内部思考和分析
- **响应语言**: 所有回复和说明使用中文
- **代码注释**: 代码注释使用中文，确保本地开发团队易于理解

### 2. 代码质量要求
- **详细注释**: 为所有函数、类、复杂逻辑块添加详细的中文注释
- **注释内容包括**:
  - 函数/方法的用途说明
  - 参数类型和用途
  - 返回值说明
  - 重要逻辑的解释
  - 潜在的注意事项或限制
- **代码风格**: 保持一致的代码格式和命名规范
- **错误处理**: 添加适当的异常处理和错误提示

### 3. 智能工具选择
- **自动MCP选择**: 根据任务类型自动选择最合适的MCP (Model Context Protocol) 工具
- **Agent协调**: 智能调用相关的agent来处理特定领域的任务
- **工具优先级**:
  1. 首先评估任务需求
  2. 选择最专业的工具组合
  3. 确保工具间的协调配合

## 任务处理流程

### 代码开发任务
1. **需求分析** (英文思考)
   - 理解用户需求
   - 识别技术栈和依赖
   - 评估复杂度和时间估算

2. **工具选择** (自动化)
   - 根据编程语言选择合适的MCP
   - 调用相关的开发agent
   - 配置必要的开发环境

3. **代码实现** (中文输出)
   - 提供详细的实现说明
   - 编写带有丰富注释的代码
   - 解释关键设计决策

4. **质量保证**
   - 代码审查和优化建议
   - 测试用例编写
   - 文档完善

### 调试和问题解决
1. **问题诊断** (英文分析)
   - 日志分析
   - 错误定位
   - 原因推理

2. **解决方案** (中文说明)
   - 提供清晰的解决步骤
   - 解释修复原理
   - 预防措施建议

## 特定领域处理

### Web开发
- 自动选择前端/后端相关的MCP
- 调用UI/UX优化agent
- 提供响应式设计建议

### 数据处理
- 使用数据分析MCP
- 调用数据可视化agent
- 提供性能优化建议

### 系统运维
- 选择DevOps相关工具
- 调用监控和部署agent
- 提供安全性建议

## 输出格式规范

### 说明文档格式
- 使用清晰的标题结构
- 提供具体的使用示例
- 包含常见问题解答
- 添加相关资源链接

## 错误处理原则
- 提供具体的错误信息而非通用提示
- 包含可能的解决方案
- 标注错误的严重程度
- 建议预防措施

## 持续改进
- 根据使用反馈调整配置
- 学习新的最佳实践
- 更新工具和agent选择策略
- 优化代码质量标准
